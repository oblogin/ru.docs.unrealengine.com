--- title: Использование компонентов permalink: /Manual/UsingComponents.html ---

Компоненты - функциональные части любого игрового объекта, при помощи которых задаются свойства и поведение для игровых объектов. Если вы еще не поняли взаимосвязь между компонентами и игровыми объектами, прочтите страницу Игровые Объекты, прежде чем двигаться дальше.

Игровой объект является контейнером для множества различных компонентов. По умолчанию все игровые объекты автоматически содержат компонент Transform. Это происходит потому, что Transform определяет положение игрового объект в пространстве его поворот и масштаб. Без компонента Transform у игрового объекта не будет своего местоположения в пространстве. В качестве примера попробуйте создать пустой игровой объект. Выберите пункт Menu GameObject -> Create Empty. Выберите новый игровой объект и посмотрите в инспекторе.

Помните, что вы всегда можете использовать Inspector, чтобы увидеть, какие компоненты присоединены к выбранному игровому объекту. По мере добавления и удаления компонентов Inspector всегда будет показывать, какие из них присоединены в данный момент. С помощью Inspector вы сможете изменить все свойства любого компонента (включая скрипты).

Добавление компонентов

Вы можете добавить компоненты к выбранному игровому объекту через Menu Components. Мы попробуем сделать это сейчас, добавив компонент Rigidbody к пустому игровому объекту, который мы только что создали. Выберите в Menu Component -> Physics -> Rigidbody. После этого в Inspector появятся свойства Rigidbody. Если вы нажмете кнопку Play, когда пустой игровой объект все еще выбран, вас может поджидать небольшой сюрприз. Попробуйте сделать это и обратите внимание на то, как Rigidbody добавил функциональность пустому игровому объекту. Position оси Y компонента Transform игрового объекта начинает уменьшаться. Это происходит потому, что физический движок Unity заставляет игровой объект падать под действием силы тяжести.

Другим вариантом является использование Component Browser, который можно активировать с помощью кнопки Add Component в окне Inspector.

Component Browser позволяет удобно перемещаться по компонентам группируя их по категориям, а также поиск который можно использовать для поиска компонентов по имени.

К одному игровому объекту можно присоединить любое количество компонентов. Некоторые компоненты лучше всего работают в сочетании с другими. Например, Rigidbody работает с любым Collider. Rigidbody управляет компонентом Transform с помощью физического движка NVIDIA PhysX, а Collider позволяет Rigidbody сталкиваться и взаимодействовать с другими Colliders.

Если вы хотите узнать больше об использовании того или иного компонента, вы можете прочитать о любом из них в справочнике компонентов. Вы также можете перейти к странице компонента из Unity, нажав на маленький значок ? на заголовке компонента в Inspector.

Редактирование компонентов

Одним из важнейших аспектов компонентов является гибкость. Когда вы присоединяете Компонент к игровому объекту, в игровом объекте появляется Компонент с различными свойствами и значениями, которые можно изменять в редакторе во время создания игры или с помощью скриптов во время игры после ее запуска. Существует два основных типа свойств: Values (Значения) и References (Ссылки).

Посмотрите на изображение ниже. Это пустой игровой объект с компонентом Audio Source. Все значения компонента Audio Source в Inspector являются значениями по умолчанию.

Этот компонент содержит одно свойство типа Reference и семь свойств типа Value. Audio Clip - это свойство типа Reference. Когда источник звука начнет воспроизведение, он попытается воспроизвести аудио файл, на который есть ссылка в свойстве Audio Clip. Если ссылка отсутствует, произойдет ошибка, так как аудио файл не будет воспроизведен. Вы должны указать ссылку на файл в свойстве Audio Clip. Для этого достаточно перетащить аудио файл из Project Window на свойство Reference или использовать Object Selector.

Компоненты могут включать ссылки на любые другие типы: Компоненты, Игровые Объекты, Активы. Подробнее о назначении ссылок вы можете прочитать на странице редактирования свойств.

Все остальные свойства Audio Clip - это свойства типа Value. Их можно настроить непосредственно в Inspector. Все свойства типа Value в Audio Clip - это переключатели, числовые значения, выпадающие поля но свойства типа Value могут быть также текстовыми строками, цветами, кривыми и другими типами. Подробнее о них и о редактировании свойств значений вы можете прочитать на странице о редактировании свойств значений.

Menu команд Component Context

Mеню Component Context содержит ряд полезных команд.

Эти же команды доступны из "кебаб" Menu (3 вертикальные точки) в крайнем правом верхнем углу панели компонента в инспекторе.

Reset

Команда Reset (Сброс) восстанавливает значения, которые свойства компонента имели до последнего сеанса редактирования.

Remove

Команда Remove Component (Удалить компонент) доступна для случаев, когда вам больше не нужен компонент, присоединенный к игровому объекту. Обратите внимание, что существуют некоторые комбинации компонентов, которые зависят друг от друга например, компонент Hinge Joint работает только при наличии компонента Rigidbody. Вы увидите предупреждение, если попытаетесь удалить компоненты, от которых зависят другие компоненты.

Move Up/Down

Команды Move Up и Move Down используются для изменения порядка расположения компонентов игрового объекта в окне Inspector.

Совет: В качестве альтернативы вы можете менять порядок компоненты вверх или вниз простым перетаскивание компонентов при помощи мышки в окне Inspector.

Copy/Paste

Команда Copy Component компонент сохраняет текущий компонент со всеми настройками свойств. Затем можно перенести сохраненные свойства компонента в другой компонент того же типа с помощью команды Paste Component Values. Вы также можете создать новый компонент со скопированными значениями на игровом объекте с помощью команды Paste Component As New.

Тестирование свойств

Пока ваша игра находится в режиме Play Mode, вы можете свободно изменять свойства любого игрового объекта в Inspector. Например, вы можете захотеть поэкспериментировать с разной высотой прыжка. Если вы создадите в скрипте свойство Jump Height, вы можете войти в режим игры, изменить значение и нажать кнопку прыжка, чтобы посмотреть, что произойдет. Затем, не выходя из Play Mode, вы можете изменить его снова и увидеть результаты в течение нескольких секунд. Когда вы выйдете из режима воспроизведения, ваши свойства вернутся к своим значениям до режима воспроизведения, так что вы не потеряете никакой работы. Этот рабочий процесс дает вам невероятную возможность экспериментировать, корректировать и совершенствовать свой игровой процесс, не затрачивая много времени на повторы. Попробуйте это с любым свойством в режиме Play Mode. Мы думаем, вы будете впечатлены.
