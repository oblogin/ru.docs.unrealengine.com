--- title: Текстуры permalink: /manual/graphics/textures ---

Как правило, сетка объекта дает лишь грубое приближение к форме, в то время как большинство мелких деталей обеспечивается Текстурами. Текстура - это обычное растровое изображение, которое накладывается на поверхность сетки. Вы можете представить себе текстуру как изображение напечатанное на резиновом листе которое растягивается и прикрепляется в соответствующих местах. Позиционирование текстуры осуществляется с помощью программного обеспечения для 3D-моделирования, которое используется для создания сеток.

Unity может импортировать текстуры из большинства распространенных форматов файлов изображений.

Текстуры для использования на 3D-моделях

Текстуры применяются к объектам с помощью Материалов. Материалы используют специализированные графические программы, называемые Шейдерами для отрисовки текстуры на поверхности сетки. Шейдеры могут реализовывать эффекты освещения и окраски для имитации блестящих или бугристых поверхностей, а также многое другое. Материалы также могут использовать более одной текстуры одновременно, комбинируя их для большей гибкости.

Текстуры должны иметь размеры, равные степени двойки (например, 32x32, 64x64, 128x128, 256x256 и т.д.). Достаточно просто поместить их в папку Assets проекта и они появятся в Project Window.

После того как текстура импортирована, ее следует назначить Материалу. Затем материал можно применить к Сетке или Системе Частиц. Используя настройки импорта, его также можно преобразовать в карту кубитов или карту нормалей. Более подробную информацию об импорте текстур можно найти на странице Компонента Текстуры.

Двухмерная графика

В двухмерных играх спрайты реализуются с помощью текстур, нанесенных на плоские сетки, которые аппроксимируют формы объектов.

Объект в двухмерной игре может потребовать набора связанных графических изображений для представления кадров анимации или различных состояний персонажа. Существуют специальные методы, позволяющие эффективно создавать и визуализировать такие наборы изображений. Дополнительную информацию см. на странице руководства о редакторе спрайтов.

GUI

Графический интерфейс пользователя (GUI) игры состоит из графики, которая не используется непосредственно в игровой сцене. но используются для того, чтобы игрок мог делать выбор и видеть информацию. Например, отображение счета и меню опций являются типичными примерами игрового GUI. Эта графика сильно отличается от той, что используется для детализации поверхности сетки, но тем не менее, она обрабатывается с помощью стандартных текстур Unity. Дополнительные сведения о системе графического интерфейса Unity см. в главе руководства "Руководство по созданию скриптов графического интерфейса".

Частицы

Сетки идеально подходят для представления твердых объектов, но менее пригодны для таких вещей, как пламя, дым и искры, оставленные магическим заклинанием. С этим типом эффектов гораздо лучше справляются системы частиц. Частица - это маленькая 2D-графика, представляющая небольшую часть чего-то жидкого или газообразного, например, облака дыма. Если создать сразу много таких частиц и привести их в движение, по желанию со случайными вариациями, они могут создать очень убедительный эффект. Например, можно изобразить взрыв, посылая частицы с текстурой огня с большой скоростью из центральной точки. Водопад может быть смоделирован путем ускорения частиц воды вниз с линии, расположенной высоко в сцене.

Системы частиц в Unity имеют богатые возможности для создания всевозможных эффектов жидкости. Дополнительную информацию см. в главе руководства по этому вопросу.

Карты высот рельефа

Текстуры можно использовать даже в тех случаях, когда изображение вообще не будет просматриваться, по крайней мере, напрямую. В серошкальном изображении каждый пиксель значение - это просто число, соответствующее оттенку серого в данной точке изображения (это может быть значение в диапазоне 0...1, где ноль - черный, а единица - белый). Хотя такое изображение можно просматривать, нет причин, почему числовые значения пикселей нельзя использовать и для других целей, и именно это делается с картами высот рельефа.

Рельеф - это сетка, представляющая область земли, где каждая точка на земле имеет определенную высоту относительно базовой линии. Карта высот для местности хранит числовые выборки высоты через регулярные интервалы в виде полутоновых значений на изображении, где каждый пиксель соответствует координатной сетке на местности. Эти значения не отображаются в сцене в виде изображения, но преобразуются в координаты, которые используются для создания сетки местности.

Интересно, что даже если карта высот не рассматривается непосредственно как изображение, все же существуют обычные методы обработки изображений, которые полезно применять к данным о высоте. Например, добавление шума к карте высот создает впечатление скалистой местности, а размытие сглаживает ее, создавая более мягкий, покатый ландшафт.

Более подробную информацию о ландшафтах в Unity можно найти в этом разделе руководства.
