--- title: Импортирование текстур permalink: /manual/graphics/textures/importing-textures ---

Текстура это растровое изображение. Текстуры можно создавать в приложении для создания цифрового контента, например Photoshop и импортировать их в Unity.

В 3D-проекте Unity импортирует файлы изображений в папку Assets как Текстуры. В 2D-проекте Unity импортирует файлы изображений в папку Assets как спрайты.

Если изображение соответствует заданным требованиям к размеру, Unity импортирует и оптимизирует его для использования в игре. Это распространяется и на многослойные файлы Photoshop PSD или TIFF.

Импорт файлов изображений в качестве текстур и спрайтов в Unity:

  1. Выберите файл изображения в Project Window.
  2. В окне Inspector установите параметры импорта текстуры.
  3. Нажмите кнопку Применить (Apply), чтобы сохранить изменения.
  4. Чтобы использовать импортированные ассеты в проекте:
    • Для 3D-проектов создайте материал и назначьте текстуру новому материалу.
    • Для 2D-проектов используйте редактор спрайтов.

Рекомендации и ограничения

В следующих темах даются рекомендации и подробно описываются любые ограничения на импорт Текстур:

HDR-текстуры

При импорте из EXR или HDR файла, содержащего информацию о HDR, программа импорта текстур автоматически выбирает подходящий формат HDR для выходной текстуры. Этот формат меняется автоматически в зависимости от того, для какой платформы вы создаете текстуру.

Размеры текстуры

В идеале размеры текстуры должны быть равны двум с каждой стороны (то есть 2, 4, 8, 16, 32, 64, 128, 256, 512, 1024, 2048 пикселей (px) и так далее). Текстуры не обязательно должны быть квадратными; ширина может отличаться от высоты.

В Unity можно использовать размеры текстур NPOT (не более двух). Однако размеры текстур NPOT обычно занимают немного больше памяти и могут быть медленнее для GPU, поэтому для производительности лучше использовать размеры, равные двум.

Если платформа или GPU не поддерживают размеры текстур NPOT, Unity масштабирует и подгоняет текстуру до размера, равного степени двойки. Этот процесс использует больше памяти и делает загрузку медленнее (особенно на старых мобильных устройствах). В целом, размеры NPOT следует использовать только для графического интерфейса.

Вы можете увеличить масштаб NPOT-текстур во время импорта, используя опцию Non Power of 2 в разделе Advanced (Дополнительно) программы импорта текстур.

Примечание: Определенные платформы могут устанавливать максимальные размеры текстур. Для DirectX максимальные размеры текстур для различных уровней возможностей следующие:

Graphics APIs / Feature levels Maximum 2D and Cubemap texture dimension size (px)
DX9 Shader Model 2 (PC GPUs before 2004) / OpenGL ES 2.0 2048
DX9 Shader Model 3 (PC GPUs before 2006) / Windows Phone DX11 9.3 level / OpenGL ES 3.0 4096
DX10 Shader Model 4 / GL3 (PC GPUs before 2007) / OpenGL ES 3.1 8192
DX11 Shader Model 5 / GL4 (PC GPUs since 2008) 16384

Примечания:

Импорт текстур позволяет выбирать размеры только до 16K (то есть 8192 x 8192 px).

Большинство графических процессоров Mali поддерживают размеры текстур до 4K для кубических карт.

Мипмапы

Мипмапы - это списки постепенно уменьшающихся версий изображения. Когда текстура использует мипмап, Unity автоматически использует меньшую версию текстуры, когда она находится далеко от камеры. Это снижает затраты производительности на визуализацию текстуры без заметной потери детализации. Мипмапы также могут уменьшить сглаживание и мерцание текстуры.

Включение мипмапов использует на 33% больше памяти, поэтому его следует использовать только для текстур, расстояние которых от камеры будет меняться. Не используйте его на текстурах, расстояние до которых не будет меняться, например, на текстурах, используемых для пользовательского интерфейса, скайбоксов и т.д.

Вы можете управлять тем, как Unity загружает мипмапы во время выполнения, используя Mip Map Streaming.

Карты нормалей

Карты нормалей используются шейдерами нормалей чтобы низкополигональные модели выглядели так, как будто они содержат больше деталей. Unity использует карты нормалей, закодированные как RGB-изображения. У вас также есть возможность генерировать карту нормалей из изображения карты высот в градациях серого.

Альфа карты

Альфа карта - это текстура, содержащая только альфа информацию. Альфа карта может использоваться для применения различных уровней прозрачности к материалу.

Альфа карта может быть создана путем создания текстуры с информацией в альфа канале или путем создания текстуры с оттенками серого и преобразования значений оттенков серого в альфа-канал в Unity.

Дополнительные сведения см. в документации по настройке импорта текстуры Alpha Source Texture.

Карты деталей

Если вы хотите создать рельеф то обычно вы используете основную текстуру для отображения участков местности, таких как трава, камни и песок. Если ваш рельеф большой, он может получиться очень размытым. Текстуры деталей скрывают этот факт, затемняя мелкие детали по мере приближения основной текстуры.

При рисовании текстур деталей нейтральный серый цвет является невидимым, белый делает основную текстуру в два раза ярче, а черный делает основную текстуру полностью черной.

Дополнительные сведения см. в документации по вторичным картам (детализированным картам).

Карты отражений (кубические карты)

Чтобы использовать текстуру для карт отражений (например, в зонде отражения или скайбоксе с кубической картой), установите для текстуры Shape в значение Cube. Дополнительные сведения см. в документации по текстурам кубических карт.
